### 8. exop-cpu main.cpp Redis+shm 重写 ✅

基于 `exop-metal/src/client/main.cpp` 模式，将 exop-cpu 从旧架构（UDP + MemBase + TF）完全重写为 Redis+shm 架构：

| 维度 | 旧架构 | 新架构 |
|------|--------|--------|
| 通信 | UDP Socket (port 8080) | Redis BLPOP (`cmd:op-cpu:0` + `sys:cmd:op-cpu:0`) |
| 数据访问 | MemBase → Tensor\<T\> | POSIX shm (mmap) → raw pointer |
| 算子调度 | TfFactory → Dispatcher | 内联模板化 C++ 循环 + dtype 动态分发 |
| 实例注册 | 无 | `/sys/op-plat/op-cpu:0` + `/op/op-cpu/list` |
| 心跳 | 无 | `/sys/heartbeat/op-cpu:0` (每 2s) |
| 退出 | 无限循环 | 响应 `sys:cmd` 的 `{"cmd":"shutdown"}` |

| 类别 | 算子 | 状态 |

| scalar | addscalar, subscalar, mulscalar, divscalar, maxscalar, minscalar, powscalar, rsubscalar, rdivscalar, rpowscalar | ✅ |

### 10. reduce CPU kernel 实现 ✅

从 stub 升级为完整实现（+226 行），覆盖 4 个算子：

| 算子 | 功能 |
|------|------|
| `sum` | 沿指定轴求和 |
| `prod` | 沿指定轴求积 |
| `reducemax` | 沿指定轴取最大值 |
| `reducemin` | 沿指定轴取最小值 |

核心实现：
- `flat_to_coords` / `coords_to_flat` — 多维坐标 ↔ 平坦索引互转
- `reduce_output_shape` — 根据 axis + keepdims 计算输出 shape
- 递归遍历被归约维度，支持负轴索引，axis 缺省时归约全部维度
