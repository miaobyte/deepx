# 进展文档 2026-04-30

## 已完成

### 1. deepx-core 统一核心库创建 ✅

创建 `executor/deepx-core/`，整合来自 3 个源库的平台无关代码：

| 模块 | 源库 | 文件数 |
|------|------|--------|
| dtype (precision/data_category/typespec) | dxlang | 3 hpp |
| stdutil (num/string/error/fs/time/vector/print) | dxlang | 7 hpp + 3 cpp |
| shmem (shm_tensor) | common-metal | 1 h + 1 cpp |
| registry | common-metal | 1 h |
| tensor (shape/tensor/tensor_base) | old-cppcommon | 3 hpp + 1 cpp |

编译为 `libdeepx_core.a` (STATIC)，仅依赖 yaml-cpp。

### 2. op-ompsimd → exop-cpu 重命名 ✅

- 目录 `executor/op-ompsimd/` → `executor/exop-cpu/`
- 内部所有 `ompsimd` → `cpu` 引用替换（源文件、头文件、CMake、测试、Dockerfile）
- 文件重命名：`dtype_ompsimd.hpp` → `dtype_cpu.hpp`, `mem_ompsimd.hpp` → `mem_cpu.hpp`
- CMake: `add_subdirectory(../deepxcore deepxcore)` → `add_subdirectory(../deepx-core deepx-core)`
- 创建 `build.sh`
- ⚠️ main.cpp 仍为旧架构（UDP + MemBase），待后续重写为 Redis+shm 版本

### 3. heap-cpu 创建 ✅

- 基于 `heap-metal/` 创建 `executor/heap-cpu/`
- `PROG_NAME = "heap-cpu"`，注册到 `/sys/heap-plat/heap-cpu-*`
- CPU only：仅依赖 `deepx-core`，无需 common-metal HAL
- 包含 CMakeLists.txt + build.sh

### 4. common-metal 精简 ✅

- 移除内置的 shm_tensor 和 registry（已迁移至 deepx-core）
- 保留仅 metal_device（Metal HAL）
- 库目标名：`deepx_common_metal` → `deepx_metal_hal`
- 新增对 `deepx-core` 的依赖

### 5. 依赖关系更新 ✅

更新以下 CMakeLists.txt 的依赖：
- `exop-cpu`: `deepxcore` → `deepx_core`
- `op-metal`: `deepx_common_metal` → `deepx_metal_hal + deepx_core`
- `heap-metal`: `deepx_common_metal` → `deepx_metal_hal + deepx_core`

### 6. 构建系统更新 ✅

- 根 `Makefile`: 新增 `build-exop-cpu`, `build-heap-cpu` 目标
- `executor/Makefile`: 新增构建变量、目标、clean 路径
- `exop-cpu/build.sh`, `heap-cpu/build.sh` 创建

### 7. 文档更新 ✅

- `.claude/CLAUDE.md`: 更新组件表、依赖关系图、构建目标
- `doc/deepx-core/README.md`: deepx-core 参考文档
